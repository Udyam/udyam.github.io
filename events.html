Mosaic

  • Are you fascinated by the Sixth Sense Technology developed by Pranav Mistry? Have you ever fancied a computer interface that lets your gestures perform tasks ? If yes, then UDYAM’15 presents to you Mosaic, an event where you will be implementing hand gesture recognition using image processing and machine learning techniques to operate a basic calculator.

    • Each team can have atmost 3 members.
    • There will be three rounds.
    • ● Round-1: Segmentation of bare hand from the real time input image using image processing.
    • ● Round-2: Gesture Recognition ( translation of hand gestures to numbers and operations ) using machine learning algorithms.
    • ● Round-3: Operation of a basic calculator using the numbers and operations indicated by the hand gestures.
    • ● There will be a small quiz at the end of the Round-2 and Round-3

  • The points will be awarded after each round based on:

    • ● the robustness of program or the restrictions it imposes on the person making gestures.
    • ● performance in the quizzes.
    • ● proficiency of the machine learning algorithm used for training.
    • ● execution time excluding the training time
    • ● innovation involved.

I-Chip

  • Practice round:
    The project has to be designed for an eight floor elevator controller. The elevator decides moving direction by comparing request floor with current floor. In a condition that the weight has to be less than 2000 kg and door has to be closed in 3 minutes. If the weight is larger than it, the elevator will alert automatically. The Door Alert signal is normally low but goes high whenever the door has been open for more than three minutes. There is a sensor at each floor to sense whether the elevator has passed the current floor. This sensor provides the signal that encodes the floor that has been passed.The core parts of the design are shift register, three cases of elevator (normal, weight alert, over time) and the while loop when receive Request Floor.

    Final Round:
    The floating point operations have found intensive applications in the various fields due to its great dynamic range, high precision and easy operation rules. With the increasing requirements for the floating point operations for the high-speed data signal processing and the scientific operation, the requirements for the high-speed hardware floating point arithmetic units have become more and more exigent. Although high-level languages have built-in floating-point data type, it is too complex to be synthesized automatically. Hence, an IEEE 754-compliant 32-bit floating point arithmetic unit has to be designed using Verilog code for the operations of addition and subtraction.

    • ● There will be 2 rounds: Practice round and Final round.

    • ● The practice round will have a simple problem statement to give the participants an insight into the various challenges faced during design and debugging of solutions before the final round that involves more complex implementation.

    • ● Scoring: Points will be awarded for both Practice and Final round. The final result will be based on the total score obtained by a team in both the rounds.

    • Team Specification: A team can consist of a maximum of 2 members.

    • ● The design must cover all aspects of problem statement and shows correct simulation for all test cases.

    • ● Analysis of the synthesis report generated in Xilinx. Although the design syntax is correct, the code may contain constructs that cannot be synthesized or lead to poor implementation. The report of hardware used for the design implementation will be taken into account.

Commnet

  • Basic Level

    • ● Participants have to implement Orthogonal Frequency Division Multiplexing (OFDM) on the transmitter side in Matlab. A test sample of data will be provided to participants using which they have to make the required modulated signal with the parameters given at the time of main event.

    • ● Parameters will include sampling frequency, number of carriers, bandwidth and intermediate modulation scheme required to be used.

    Advance Level

    • ● Participants have to implement OFDM on the receiver side in Matlab on a real signal. A test signal will be provided to participants using which they have to find certain parameters like no. of carriers and modulation scheme used to make the test signal. After knowing these parameters, they have to demodulate a distorted transmitted signal into the real signal.
    • ● Quiz: A quiz will be conducted covering basic understanding of digital communication system, signal processing schemes and their implementation in Matlab.
    • ● There will be 2 levels: basic level and advance level

    • ● There will be quiz based on the concepts of digital communication and signal processing. The basic level will give the participants an insight into the implementation of transmitter side of a digital communication system in which participants can learn basic modulation and signal processing techniques.

    • ● Advance level includes the implementation of receiver side and training of a digital communication system where participants have to deal with real signals

    • ● Scoring: points will be awarded for both rounds and quiz. The final result will be based on total score obtained by a team in both the rounds and the quiz.

    • ● Software: students have to write their codes in Matlab

    • ● Team specifications: A team consists of maximum of 2 members

    • ● The implementation must satisfy all the parameters of the problem statement and shows correct output signals for all test cases

    • ● The quality of signals at the output and basic understanding of digital communication will be judging criteria for the event.

Krypto

  • Does hacking attract you? Do you want to know how your confidential information is kept secret from hackers?
    If your answer is YES! Then Cryptography is the field, answers you all these questions. Cryptography is the art and science of keeping information secure from unintended audiences. Conversely, cryptanalysis is the art and science of breaking encoded data.

    Udyam’15 presents you Krypto, an event, where you will learn how to protect your conversation safe from eavesdroppers and even how to break into secrets of others.

    • Qualifying Round: Participants are required to build a terminal software either in java, python or C++ to encrypt/decrypt a text file(.txt) using DES CBC mode encryption without using any existing libraries of Cryptography. No extra credits for GUI.
      ● Deadline for submission: 10th March, 2015

    • First Round: Participants are required to encrypt a file containing one/more text/image files using any authentication encryption standards. Libraries available for crypto can be used.

    • Final Round: Participants need to break the security of RSA based cryptosystem. Public key and an encrypted message will be provided, and the message have to be retrieved

  • There will be 3 rounds: Qualifying, basic level and advance level.
    Pre-events to be announced soon. It will have questions on basic concepts of Cryptography and programming tasks.

    Software: Participants can use Java, C++ or Python only.

    Team specifications: A team consists of maximum of 2 members.

    There will be a special prize for the best team from first year.

  • The teams will be judged on the following criteria:

    • ● Time taken to encrypt/decrypt.
    • ● Semantic Security of the cipher.

Funckit

  • In a Airport it is necessary for a automated system to designate a gate for incoming flights, in this way it is also helpful for passengers to know about at which gate flight is present. In a airport prototype there are 4 gates and and day starts from 0 hrs to 16 hrs.
    Design a circuit with 4 ports and which will automatically designate the gate for a flight .
    Designate a led at each gate show whether a flight is present at that particular gate in the circuitry. Also designate a 2 bit counter at each gate which shows how much time left for the flight to take off.

    There are two types of flights 1.International 2.Domestic.
    International flight stays for 2 hours in the Airport and Domestic stays for 1 hour.
    International Flight prefers gate 3,if gate 3 is not empty international flight can go into any of the other empty gates. Domestic flights are not supposed to go into the gate number 3.

  • There are two rounds,in the first round simulation of circuit in Proteus for any random schedule.In 2nd round the hardware implementation must be done for a schedule which will be given a day before the event

  • The implementation must satisfy all the above conditions and show correct cases for different schedules of flights.
    Team Strength: 3 Members
